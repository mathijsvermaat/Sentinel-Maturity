## Table Formats

**Tables and Rationale** columns: `| Table | Description | Retention Recommendation | Rationale | Forensic Value | Example Detection |`
- Retention default: `Analytics: 90d / Lake: 365d`
- Forensic Value: action-oriented phrasing ("ReconstructÔÇŽ", "ProveÔÇŽ", "DetectÔÇŽ")
- Table names in backticks

**Per-event-ID tier tables** (when a connector page enumerates individual Event IDs under "Key Events by Tier" or equivalent) columns: `| Event ID | Description | Minimal | Common | Full | Rationale | Forensic Value | MITRE | Example Detection |`
- The `Rationale` and `Example Detection` cells **must not** contain MCSB control IDs (e.g. `MCSB IM-1.`, `MCSB PA-1.`, `MCSB LT-3.`) or MITRE technique IDs (e.g. `MITRE T1136.001.`, trailing `(T1110)`, `(T1059.001)`). MCSB belongs in the `MCSB Control Mapping` section; MITRE technique/strategy references belong in the new `MITRE` column and the dedicated `MITRE Detection Strategies` section.
- The `MITRE` column links to the page's own `## MITRE Detection Strategies` section using `[N mappings](#mitre-detection-strategies)` (or `[1 mapping](#mitre-detection-strategies)`), where `N` is the count of distinct `(CurrentTechId, DetId)` pairs in `C:\Temp\tech_to_det_v3_detail.csv` whose `Platform` = the connector's platform and whose `EventCode` field contains the row's Event ID (regex `\bID\b`, so comma-separated lists like `EventCode=4624, 4648` match both). When no row matches, render `ÔÇö`. Do **not** inline individual technique or strategy links into the per-event cell ÔÇö the strategies section is the canonical list.

**Example Detections** columns: `| Detection | Table(s) / Event ID(s) | MITRE ATT&CK | Detection Strategy | Description |`
- MITRE format: full technique ID `T1110.003` linked to `https://attack.mitre.org/techniques/T1110/003/`
- Detection Strategy format: `[DET####](https://attack.mitre.org/detectionstrategies/DET####/) ÔÇö [Strategy name]`, or `ÔÇö *(no published strategy)*` if none

**MITRE Detection Strategies** columns: `| Technique | Detection Strategy | MITRE Log Sources (<Platform>) |`
- Source of truth: `C:\Temp\tech_to_det_v3_detail.csv` (one row per `CitedTech, Strategy, Analytic, LogSource`; built from the MITRE STIX `enterprise-attack.json` bundle by `C:\Temp\build_mitre_map_v3.ps1`; follows `revoked-by` chain). Summary file: `C:\Temp\tech_to_det_v3.csv`. The older `*_v2.csv` files are superseded.
- The third column header substitutes the connector's primary platform (e.g. `(Windows)` for Windows event sources, `(Linux)` for syslog, `(Azure)` for cloud activity).
- Filter the detail CSV by `CitedTechId`, `DetId` and `Platform = <connector's platform>`. Group rows by `Channel` and concatenate `EventCode` values within each group. Render `` `Channel`: <codes> `` joined with ` ┬Ě `. Example: `` `Security`: 4624, 4648 ┬Ě `Sysmon`: 1, 3, 22 ``.
- Use the literal `Channel` and `EventCode` strings from the CSV (they come directly from MITRE's `x_mitre_log_source_references`). **Never** substitute "similar-looking" event IDs from the connector page's own Event-ID tables.
- When no row matches the connector's platform, render: `*MITRE has not published a <Platform> analytic for this strategy*`.
- When the platform rows contain only non-`Security` channels (only `Sysmon` / `PowerShell` / `System`), still list them verbatim and append `*(no \`Security\`-channel analytic published)*`.
- If a cited technique was revoked, annotate as `Txxxx.xxx ÔÇö [Name] *(revoked Ôćĺ Tyyyy.yyy)*` and link the strategy attached to the current technique.
- Include a `[!NOTE]` "log sources are verbatim from MITRE" callout in the section so readers know the third column is not author-curated.
- Include a `[!NOTE]` legacy-ID callout only when at least one row uses a revoked ID.
- Only mark a technique as "no published strategy" (in the `Example Detections` table) if neither the legacy nor the current (post-`revoked-by`) technique appears in `tech_to_det_v3.csv` at all.

**MCSB Control Mapping** columns: `| MCSB Control | Relevance |`

**Tools** columns: `| Tool | Type | Purpose | Source | Guide |`
