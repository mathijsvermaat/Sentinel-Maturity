# Connector Page Rules

Use `connectors/_TEMPLATE.md` as the structural reference.

## Section Order

1. H1 title → Metadata line (Tier · Type · Free ingestion) → Contents → Overview → Licensing Benefits → Tables and Rationale → Example Detections → MCSB Control Mapping → [Optional sections] → Notes → Tools → References → Footer

## Table Formats

**Tables and Rationale** columns: `| Table | Description | Retention Recommendation | Rationale | Forensic Value | Example Detection |`
- Retention default: `Analytics: 90d / Lake: 365d`
- Forensic Value: action-oriented phrasing ("Reconstruct…", "Prove…", "Detect…")
- Table names in backticks

**Example Detections** columns: `| Detection | Table(s) / Event ID(s) | MITRE ATT&CK | Description |`
- MITRE format: full technique ID `T1110.003`

**MCSB Control Mapping** columns: `| MCSB Control | Relevance |`

**Tools** columns: `| Tool | Type | Purpose | Source | Guide |`

## Footer

Always end with: `[← Back to Connectors](README.md) · [← Back to Sentinel Maturity Model](../README.md)`
