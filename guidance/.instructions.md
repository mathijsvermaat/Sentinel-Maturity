# Guidance Page Rules

## Section Order

1. H1 title → Contents → Overview (may include blockquote thesis) → 3–5 H2 content sections → References → Footer

## Writing Style

- Explanatory narrative, not procedural
- Tie major points to MCSB, NIST, CIS, NIS2, GDPR with specific control IDs
- Use comparison tables for framework mappings
- Use callout blocks: `> [!NOTE]`, `> [!IMPORTANT]`, `> [!TIP]`, `> [!WARNING]`
- Reference connectors with relative links: `[Name](../connectors/file.md)`

## Footer

Always end with: `[← Back to Guidance](README.md) · [← Back to Sentinel Maturity Model](../README.md)`
